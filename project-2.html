PROJECT LAPORAN PRAKTIKUM 2

Laporan ini membahas WinBox, aplikasi grafis untuk mengelola perangkat MikroTik RouterOS. MikroTik dapat dikonfigurasi dalam mode router atau bridge untuk mengatur jaringan. Laporan ini juga menjelaskan langkah konfigurasi dasar router MikroTik RB931-2nD, termasuk pengaturan IP, DHCP, NAT, dan DNS untuk distribusi koneksi internet.

Konfigurasi MikroTik dengan WinBox

A. Pengenalan WinBox

1. WinBox adalah sebuah aplikasi grafis yang dirancang oleh MikroTik untuk memungkinkan pengelolaan dan konfigurasi perangkat RouterOS dengan antarmuka pengguna yang intuitif dan responsif. Aplikasi ini memudahkan administrator jaringan dalam mengelola perangkat MikroTik tanpa harus menggunakan perintah berbasis teks (command line interface atau CLI).

2. Meskipun dikembangkan secara native untuk sistem operasi Windows (Win32/Win64), WinBox juga dapat dijalankan pada sistem operasi lain seperti macOS dan Linux melalui emulator Wine. Aplikasi ini menyediakan akses lengkap ke semua fitur RouterOS, termasuk routing, manajemen firewall, pengaturan bandwidth, konfigurasi hotspot, dan manajemen titik akses nirkabel (Wi-Fi).

3. Keunggulan WinBox

  • 1. Antarmuka grafis yang sederhana Memungkinkan konfigurasi cepat tanpa perlu menghafal perintah CLI.
  • 2. Fleksibel Dapat diakses dari berbagai platform dengan bantuan emulator.
  • 3. Real-time monitoring Menyediakan informasi dan statistik jaringan secara langsung.
  • 4. Multifungsi Mengelola semua aspek router seperti DHCP server, NAT, firewall, dan lain-lain dengan mudah.

4. Dengan WinBox, konfigurasi yang rumit dapat dilakukan dengan lebih mudah, terutama bagi pengguna yang tidak terbiasa dengan antarmuka teks.

B. Topologi Jaringan dan Mode Operasi MikroTik

1. MikroTik RouterOS menawarkan fleksibilitas tinggi dalam pengelolaan jaringan, memungkinkan pengguna untuk mengkonfigurasinya sebagai router atau bridge. Setiap mode ini memiliki fungsi dan kegunaan yang berbeda sesuai dengan kebutuhan jaringan.

  • 1. Mode Router MikroTik bertindak sebagai perangkat router yang mengarahkan lalu lintas antar jaringan yang berbeda. Mode ini ideal untuk menghubungkan jaringan lokal (LAN) dengan internet atau jaringan lain.
  • 2. Mode Bridge (Switch) Mode bridge digunakan ketika MikroTik ingin difungsikan sebagai switch, di mana beberapa perangkat di jaringan lokal (LAN) dapat terhubung dalam satu jaringan yang sama.

2. Contoh Kasus Penggunaan

  • 1. Mode Router Digunakan saat ingin menghubungkan jaringan internal ke internet, menyediakan fitur firewall untuk keamanan, dan mengatur distribusi IP melalui DHCP.
  • 2. Mode Bridge Cocok untuk menyatukan beberapa jaringan di dalam satu jaringan logis, seperti menyatukan jaringan kantor dengan jaringan Wi-Fi publik.

C. Langkah-Langkah Konfigurasi MikroTik dalam Mode Router

Berikut adalah prosedur untuk mengkonfigurasi MikroTik RB931-2nD sebagai router dengan alamat IP lokal 192.168.1.40/24. Konfigurasi ini akan membuat perangkat dapat mendistribusikan koneksi internet kepada klien di jaringan lokal.

  1. 1. Menghubungkan MikroTik ke PC
    Hubungkan router MikroTik ke komputer menggunakan kabel Ethernet. Pada perangkat RB931-2nD, pilih salah satu port dari port 2 hingga port 5. Setelah terhubung, pastikan komputer berada dalam satu subnet default MikroTik agar WinBox dapat mendeteksi perangkat.
  2. 2. Mengunduh dan Membuka WinBox
    - Unduh WinBox dari situs resmi MikroTik di https://mikrotik.com/download.
    - Buka aplikasi WinBox, lalu navigasi ke tab “Neighbors”. WinBox akan mendeteksi router MikroTik yang terhubung melalui alamat MAC-nya.
    - Pilih alamat MAC tersebut, masukkan username default (biasanya "admin" tanpa password), dan login.
  3. 3. Menghapus Konfigurasi Default
    Untuk memastikan konfigurasi yang bersih tanpa pengaturan lama yang dapat menyebabkan konflik, pilih opsi Remove Configuration di antarmuka WinBox. Langkah ini akan menghapus semua pengaturan default dan memungkinkan memulai konfigurasi dari awal.
  4. 4. Menetapkan Alamat IP untuk LAN
    - Buka menu IP > Addresses.
    - Klik tombol “+” untuk menambahkan alamat IP baru. Masukkan alamat IP yang diinginkan untuk LAN, misalnya 192.168.1.40/24.
  5. 5. Mengkonfigurasi DHCP Server
    Untuk memungkinkan perangkat dalam jaringan mendapatkan IP secara otomatis, buka menu IP > DHCP Server dan klik “+” untuk menambahkan DHCP Server baru. Tentukan alamat gateway sebagai 192.168.1.40 dan rentang IP yang akan diberikan, misalnya 192.168.1.50 hingga 192.168.1.100.
  6. 6. Menambahkan NAT untuk Akses Internet
    Agar perangkat dalam jaringan lokal dapat mengakses internet melalui alamat IP publik, buka menu IP > Firewall, lalu masuk ke tab NAT. Klik “+” untuk menambahkan aturan NAT baru. Pada tab General, pilih “srcnat” pada chain. Pada tab Action, pilih “masquerade”.
  7. 7. Menyetel DNS Server
    DNS diperlukan agar perangkat dapat mengakses internet dengan nama domain. Buka menu IP > DNS dan tambahkan alamat DNS server, misalnya 8.8.8.8 (Google DNS).
  8. 8. Menyimpan Pengaturan dan Reboot
    Setelah semua konfigurasi selesai, klik Apply dan OK untuk menyimpan pengaturan. Reboot perangkat MikroTik melalui menu System > Reboot.

D. Pengujian dan Troubleshooting

Setelah proses reboot selesai, lakukan pengujian dengan memastikan perangkat yang terhubung ke jaringan lokal dapat:

  • 1. Mendapatkan alamat IP secara otomatis dari DHCP server.
  • 2. Mengakses internet dengan alamat IP yang diberikan.
  • 3. Melakukan ping ke perangkat lain dalam jaringan lokal dan ke internet.

Jika terdapat masalah seperti perangkat tidak mendapatkan IP atau tidak bisa mengakses internet, beberapa langkah troubleshoot yang dapat dilakukan meliputi:

  • 1. Memeriksa koneksi fisik dan memastikan kabel Ethernet terhubung dengan baik.
  • 2. Memverifikasi bahwa DHCP Server berjalan dengan benar.
  • 3. Memastikan aturan NAT telah diaktifkan dan diterapkan pada interface yang benar.

Dengan mengikuti langkah-langkah di atas, MikroTik RB931-2nD akan berfungsi sebagai router yang siap untuk mendistribusikan koneksi internet ke perangkat-perangkat di jaringan lokal.
