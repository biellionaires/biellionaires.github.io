PROJECT LAPORAN PRAKTIKUM 5

Laporan ini membahas konsep dan fungsi NAT (Network Address Translation) dalam pengaturan jaringan, termasuk peran NAT dalam firewall dan bridge. Proses ini mencakup pengertian NAT firewall dan NAT bridge, serta perbedaan keduanya dalam pengelolaan alamat IP dan keamanan jaringan. Selain itu, laporan ini menjelaskan tentang AP Bridge dan Bridge, yang menguraikan fungsinya dalam menghubungkan jaringan kabel dan nirkabel serta segmentasi jaringan di layer 2. Dengan memahami topik ini, pembaca dapat memperoleh wawasan tentang bagaimana NAT dan bridge berkontribusi pada manajemen dan keamanan jaringan secara keseluruhan.

Pengertian NAT Bridge dan NAT Firewall, AP Bridge dan Bridge

1. NAT (Network Address Translation)

NAT (Network Address Translation) adalah teknik jaringan yang digunakan untuk menerjemahkan alamat IP dalam suatu jaringan. Teknik ini sering digunakan untuk menyembunyikan alamat IP privat di belakang satu atau beberapa alamat IP publik. Dengan mengimplementasikan NAT, banyak perangkat di jaringan lokal dapat berbagi satu alamat IP publik ketika mengakses internet. Ini tidak hanya membantu mengurangi konsumsi alamat IP publik yang semakin terbatas, tetapi juga meningkatkan keamanan jaringan dengan menyembunyikan struktur internal dari jaringan lokal.

2. Jenis-Jenis NAT

  • Static NAT: Menghubungkan satu alamat IP publik dengan satu alamat IP privat.
  • Dynamic NAT: Menghubungkan banyak alamat IP privat ke sekelompok alamat IP publik.
  • Port Address Translation (PAT): Memungkinkan beberapa alamat IP privat untuk terhubung ke satu alamat IP publik dengan menggunakan nomor port yang berbeda.

3. NAT di Menu Firewall

Pada router atau perangkat jaringan, NAT biasanya dikonfigurasi melalui menu Firewall, karena di situlah proses filtering dan manajemen alur paket terjadi. Konfigurasi NAT di Firewall berfungsi untuk mengatur lalu lintas yang masuk dan keluar dari jaringan, serta memberikan lapisan tambahan dalam pengamanan jaringan.

Ciri dan Fungsi NAT di Menu Firewall

  • Pengaturan Routing dan Translasi: Mengatur bagaimana IP privat diterjemahkan ke IP publik saat paket melewati firewall.
  • Source NAT (Masquerade): Menyembunyikan IP privat di balik satu IP publik untuk koneksi keluar.
  • Destination NAT (DNAT): Mengalihkan paket dari luar ke alamat internal tertentu.
  • Control dan Filter Paket: Mengatur port forwarding dan QoS.
  • Keamanan: Memperkuat keamanan jaringan dengan menyembunyikan struktur internal.

Kelebihan NAT di Firewall

  • Fleksibilitas dan kontrol lebih baik terhadap traffic.
  • Mudah digunakan untuk SNAT dan DNAT.
  • Mendukung konfigurasi kompleks seperti VPN dan tunneling.

Contoh Penggunaan NAT di Firewall

  • Koneksi internet dengan banyak perangkat privat menggunakan satu alamat IP publik.
  • Mengarahkan trafik dari internet ke server internal menggunakan port forwarding.

4. NAT di Menu Bridge

Bridge bekerja pada lapisan 2 OSI (Data Link Layer), sementara NAT terkait dengan lapisan 3 (Network Layer). Oleh karena itu, NAT di menu Bridge memiliki tujuan dan mekanisme berbeda dibandingkan dengan NAT di Firewall.

Ciri dan Fungsi NAT di Menu Bridge

  • Digunakan dalam bridging dengan translasi IP minimal, seperti komunikasi antar VLAN.
  • Jarang digunakan, biasanya untuk segmentasi khusus.
  • Tidak memiliki kontrol trafik selengkap NAT di firewall.

Keterbatasan NAT di Bridge

  • Kurang fleksibel dibandingkan NAT di firewall.
  • Tidak cocok untuk jaringan skala besar.

5. AB (Access Point Bridge) vs AB (Bridge)

A. AB (Access Point Bridge)

AB adalah mode di mana Access Point memungkinkan perangkat klien terhubung secara wireless dan menghubungkan beberapa perangkat jaringan melalui akses nirkabel. Fungsi utamanya adalah mengintegrasikan jaringan kabel dan nirkabel menjadi satu kesatuan yang dapat saling berkomunikasi.

Ciri dan Fungsi AB

  • Menghubungkan jaringan wireless dan kabel.
  • Mendukung banyak klien wireless.
  • Memancarkan SSID untuk diakses oleh klien.
  • Meneruskan frame wireless ke perangkat kabel.

Kapan AB Digunakan?

Ketika perlu menghubungkan jaringan kabel dan wireless dalam satu subnet, terutama di kantor atau rumah yang memiliki perangkat wireless dan kabel.

B. AB (Bridge)

AB adalah mekanisme yang menghubungkan dua atau lebih jaringan di lapisan 2. Fungsinya adalah meneruskan frame Ethernet antar segmen jaringan tanpa mengubah alamat IP atau protokol.

Ciri dan Fungsi AB

  • Layer 2 forwarding: Meneruskan frame tanpa melihat paket layer 3.
  • Segmentasi jaringan: Memisahkan lalu lintas untuk mencegah broadcast storm.
  • Tidak memancarkan SSID.
  • Menghubungkan VLAN dalam satu jaringan.

Kapan AB Digunakan?

Dalam jaringan LAN untuk menghubungkan dua segmen fisik dan untuk segmentasi yang mencegah broadcast berlebihan.

Kesimpulan

NAT di Firewall lebih berguna untuk routing dan manajemen paket secara kompleks, menawarkan fleksibilitas dan kontrol tinggi dalam pengaturan trafik jaringan. Sementara itu, NAT di Bridge hanya digunakan dalam skenario layer 2 tertentu dan kurang populer dalam praktik jaringan. AB memungkinkan integrasi wireless dan kabel melalui Access Point, sedangkan AB bekerja di lapisan 2 untuk menghubungkan segmen jaringan kabel, membantu dalam segmentasi dan pengelolaan lalu lintas di dalam jaringan.
