PROJECT LAPORAN PRAKTIKUM 4

Laporan ini membahas langkah-langkah konfigurasi routing statis pada tiga perangkat MikroTik, yang memungkinkan setiap jaringan lokal untuk saling berkomunikasi dengan lancar. Proses ini mencakup konfigurasi IP address pada setiap router, penambahan routing statis, dan pengujian koneksi menggunakan perintah ping untuk memastikan paket data dapat berpindah antar jaringan. Selain itu, laporan ini juga menjelaskan troubleshooting dasar yang dapat dilakukan jika terjadi masalah koneksi, termasuk pengecekan IP, status interface, dan pengaturan firewall. Dengan mengikuti panduan ini, administrator dapat memastikan jaringan berjalan dengan baik dan stabil melalui routing statis, yang cocok untuk jaringan berskala kecil hingga menengah.

Langkah-Langkah Konfigurasi Routing Static 3 Mikrotik

1. Topologi dan Persiapan Awal

Sebelum masuk ke konfigurasi, kita perlu menentukan topologi jaringan dan alamat IP yang akan digunakan.

MikroTik 1 (R1):
Interface ether1: 10.10.10.1/24
Interface ether2: 192.168.100.1/24

MikroTik 2 (R2):
Interface ether1: 10.10.10.2/24
Interface ether2: 11.11.11.1/24
Interface ether3: 192.168.200.1/24

MikroTik 3 (R3):
Interface ether1: 11.11.11.2/24
Interface ether2: 192.168.50.1/24

Dengan topologi ini, kita akan mengkonfigurasi routing statis untuk memastikan setiap router dapat berkomunikasi dengan jaringan yang berbeda.

2. Langkah Konfigurasi MikroTik

A. Konfigurasi IP Address

Langkah pertama adalah mengkonfigurasi IP address pada setiap MikroTik.

  1. Langkah pada MikroTik 1 (R1):
    - Interface: ether1 | Address: 10.10.10.1/24
    - Interface: ether2 | Address: 192.168.100.1/24
  2. Langkah pada MikroTik 2 (R2):
    - Interface: ether1 | Address: 10.10.10.2/24
    - Interface: ether2 | Address: 11.11.11.1/24
    - Interface: ether3 | Address: 192.168.200.1/24
  3. Langkah pada MikroTik 3 (R3):
    - Interface: ether1 | Address: 11.11.11.2/24
    - Interface: ether2 | Address: 192.168.50.1/24

B. Konfigurasi Static Routing

Setelah konfigurasi IP, kita tambahkan routing statis agar jaringan bisa saling berkomunikasi.

  1. Konfigurasi di MikroTik 1 (R1):
    - Dst. Address: 192.168.200.0/24 | Gateway: 10.10.10.2
    - Dst. Address: 192.168.50.0/24 | Gateway: 10.10.10.2
  2. Konfigurasi di MikroTik 2 (R2):
    - Dst. Address: 192.168.100.0/24 | Gateway: 10.10.10.1
    - Dst. Address: 192.168.50.0/24 | Gateway: 11.11.11.2
  3. Konfigurasi di MikroTik 3 (R3):
    - Dst. Address: 192.168.100.0/24 | Gateway: 11.11.11.1
    - Dst. Address: 192.168.200.0/24 | Gateway: 11.11.11.1

C. Testing Koneksi Antar MikroTik

Untuk memastikan bahwa konfigurasi berhasil, lakukan pengujian koneksi dengan menggunakan perintah ping:

  1. Ping dari MikroTik 1 ke MikroTik 2: ping 192.168.200.1
  2. Ping dari MikroTik 1 ke MikroTik 3: ping 192.168.50.1

D. Troubleshooting Koneksi (Opsional)

Jika terjadi masalah koneksi, beberapa langkah yang bisa dilakukan untuk troubleshooting antara lain:

  • Periksa apakah IP address sudah dikonfigurasi dengan benar.
  • Pastikan routing statis sudah menunjuk ke gateway yang tepat.
  • Periksa status interface pada setiap router.
  • Jika firewall diaktifkan, coba nonaktifkan sementara untuk memastikan bahwa firewall bukan penyebab masalah.

3. Kesimpulan

Dengan konfigurasi yang tepat, routing statis pada tiga MikroTik memungkinkan setiap jaringan lokal berkomunikasi dengan lancar. Static routing memberikan kontrol penuh bagi administrator, namun membutuhkan perhatian dalam konfigurasi dan pemeliharaan manual. Pengujian koneksi menggunakan perintah ping membuktikan bahwa paket dapat berpindah antar jaringan dengan baik. Routing statis cocok untuk jaringan kecil hingga menengah karena tidak memerlukan overhead tambahan yang dihasilkan oleh dynamic routing.
